## Why this matters

Three dependencies are declared but unused, inherited from the next-forge template:
- `concurrently` in `apps/api` — no script or code invokes it.
- `@auto-it/first-time-contributor` in the root — a plugin for the `auto` release
  tool, but there is no `auto` dependency; the `.autorc` is itself a next-forge
  leftover (`repo: next-forge`).
- `date-fns` in `apps/app` and `packages/design-system` — neither imports it (only
  `apps/web` uses it; `react-day-picker` bundles its own copy).

Dead deps add install/audit surface and mislead contributors about what the
packages actually use. Removing them is low-risk housekeeping.
