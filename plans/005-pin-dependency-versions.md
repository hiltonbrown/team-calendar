## Why this matters

Four production dependencies are declared as `"latest"`, which makes builds non-reproducible: any fresh install (a new CI runner, a Vercel build after a registry publish) can silently pull a different — possibly breaking — major version of Inngest (the entire background-job layer) or the design system's component primitives. Separately, root `engines` declares `"node": ">=18"`; Node 18 is past end-of-life and Vercel's platform default is Node 24, so the floor documents a runtime nobody should use.

## Current state

- `"inngest": "latest"` in three manifests:
  - `packages/jobs/package.json:22`
  - `packages/availability/package.json:18`
  - `apps/api/package.json:31`
- `"radix-ui": "latest"` in `packages/design-system/package.json:34`
- Resolved versions currently locked in `bun.lock`: `inngest@4.5.0`, `radix-ui@1.4.3` (verified at planning time — re-verify in Step 1).
- Root `package.json:34`: `"engines": { "node": ">=18" }`.
- Every other dependency in the monorepo uses pinned or caret ranges (e.g. `"zod": "^4.4.3"` consistently across 15 packages), so `"latest"` is the outlier, not the convention.
- Package manager: Bun workspaces (`bun.lock` at root). `bun install` refreshes the lockfile; CI uses `bun install --frozen-lockfile`.

## Steps

### Step 1: Read the currently-locked versions

`grep -oE '"inngest@[^"]+"' bun.lock | head -1` and `grep -oE '"radix-ui@[^"]+"' bun.lock | head -1`. Use exactly these versions in Step 2 (expected: `4.5.0` and `1.4.3`; if different, use what the lockfile says — the goal is zero runtime change).

**Verify**: both greps return exactly one version each.

### Step 2: Replace the "latest" ranges

- In `packages/jobs/package.json`, `packages/availability/package.json`, `apps/api/package.json`: `"inngest": "latest"` → `"inngest": "^<locked>"` (e.g. `"^4.5.0"`). Use the identical range in all three.
- In `packages/design-system/package.json`: `"radix-ui": "latest"` → `"radix-ui": "^<locked>"` (e.g. `"^1.4.3"`).

**Verify**: `grep -rn '"latest"' apps/*/package.json packages/*/package.json` → no matches.

### Step 3: Raise the engines floor

In root `package.json`: `"engines": { "node": ">=18" }` → `"engines": { "node": ">=20" }`.

(Why >=20 not >=24: 20 is the oldest non-EOL LTS; nothing in the repo requires newer, and a conservative floor avoids breaking contributors on LTS installs. The runtime on Vercel is controlled by the platform, not this field.)

**Verify**: `grep -n '"node"' package.json` → `">=20"`.

### Step 4: Relock and confirm zero dependency drift

Run `bun install`. Then confirm the lockfile did not change resolved versions for these packages:

`grep -oE '"inngest@[^"]+"' bun.lock | head -1` → same version as Step 1; same for `radix-ui`.

Then run the full verification suite.

**Verify**: `bun install --frozen-lockfile` → exit 0 (lockfile consistent). `bun run test` → all pass. `bun run check` → exit 0.
