## Dependency notes

- All plans are independent and can be executed in any order. Recommended order
  is leverage: security (001-002), reliability bugs (003-005), then the rest.
- **006 is a soft prerequisite for the refactor-heavy plans.** It re-enables
  package-level typechecking; running it first means later plans get a real type
  gate. But it will surface latent type errors that must be cleared, so it can be
  a rabbit hole — it is sequenced as P2, not blocking.
- 013-015 (characterisation tests) should ideally land *before* the perf refactors
  (007-009) that touch the same services, so the refactors have a safety net.
  Not a hard dependency, but prefer tests-first if executing both.
- 016-018 are **design/spike plans**, not build-everything plans. Each ends with
  a written design + open questions for maintainer review, not a finished feature.
